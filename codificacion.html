Programación

Codificación del Sistema

El sistema fue programado en lenguaje C++ utilizando el entorno de desarrollo Arduino IDE, aprovechando su amplio ecosistema de librerías compatibles con el ESP32. El código integra cuatro librerías principales que trabajan en conjunto para lograr la adquisición, sincronización y almacenamiento de los datos.

📦 Librerías Utilizadas

DHT.h Maneja la comunicación con el sensor DHT22, facilitando la lectura de temperatura y humedad sin necesidad de implementar el protocolo manualmente.
SD.h Permite la lectura y escritura de archivos en la tarjeta microSD mediante el protocolo SPI, gestionando la apertura, escritura y cierre del archivo CSV.
SPI.h Habilita el bus de comunicación SPI del ESP32, necesario para el funcionamiento del módulo de tarjeta SD.
WiFi.h Gestiona la conexión inalámbrica del ESP32 a la red WiFi local para acceder al servidor de tiempo NTP.
time.h Permite obtener y formatear la hora y fecha real desde un servidor NTP en internet, garantizando marcas de tiempo precisas en cada registro.

💻 Código Fuente

// ================================================
// Sistema de Monitoreo de Temperatura y Humedad
// ESP32 + DHT22 + Módulo MicroSD
// Universidad Surcolombiana – 2026
// ================================================

#include <SPI.h>
#include <SD.h>
#include <DHT.h>
#include <WiFi.h>
#include "time.h"

// ── Pines ──────────────────────────────────────
#define SD_CS    5
#define DHTPIN   4
#define DHTTYPE  DHT22

// ── Credenciales WiFi ──────────────────────────
const char* ssid     = "TU_RED_WIFI";
const char* password = "TU_CONTRASEÑA";

// ── Servidor NTP – Colombia GMT-5 ──────────────
const char* ntpServer         = "pool.ntp.org";
const long  gmtOffset_sec     = -5 * 3600;
const int   daylightOffset_sec = 0;

DHT dht(DHTPIN, DHTTYPE);

// ── SETUP ──────────────────────────────────────
void setup() {
  Serial.begin(115200);
  delay(1000);
  dht.begin();

  // Conexión WiFi con límite de intentos
  Serial.print("Conectando WiFi");
  WiFi.begin(ssid, password);

  int intentos = 0;
  while (WiFi.status() != WL_CONNECTED && intentos < 20) {
    delay(500);
    Serial.print(".");
    intentos++;
  }

  if (WiFi.status() == WL_CONNECTED) {
    Serial.println("\nWiFi conectado");
    configTime(gmtOffset_sec, daylightOffset_sec, ntpServer);
  } else {
    Serial.println("\nSin WiFi - continuando sin hora");
  }

  // Inicializar tarjeta SD
  if (!SD.begin(SD_CS)) {
    Serial.println("Error iniciando SD");
    return;
  }
  Serial.println("SD lista");

  // Crear archivo con encabezado si no existe
  if (!SD.exists("/datos.csv")) {
    File archivo = SD.open("/datos.csv", FILE_WRITE);
    if (archivo) {
      archivo.println("Fecha,Hora,Temperatura,Humedad");
      archivo.close();
    }
  }
}

// ── LOOP ───────────────────────────────────────
void loop() {
  float temperatura = dht.readTemperature();
  float humedad     = dht.readHumidity();

  // Validar lectura del sensor
  if (isnan(temperatura) || isnan(humedad)) {
    Serial.println("Error leyendo DHT22");
    delay(2000);
    return;
  }

  // Obtener fecha y hora del servidor NTP
// Si no hay conexión, usa el reloj interno del ESP32
  struct tm timeinfo;
  if (!getLocalTime(&timeinfo)) {
    Serial.println("Sin hora NTP - usando hora interna");
  }

  char fecha[11];
  char hora[9];
  strftime(fecha, sizeof(fecha), "%Y-%m-%d", &timeinfo);
  strftime(hora,  sizeof(hora),  "%H:%M:%S", &timeinfo);

  // Mostrar en monitor serial
  Serial.printf("%s %s | %.2f °C | %.2f %%\n",
                fecha, hora, temperatura, humedad);

  // Guardar en tarjeta SD
  File archivo = SD.open("/datos.csv", FILE_APPEND);
  if (archivo) {
    archivo.printf("%s,%s,%.2f,%.2f\n",
                   fecha, hora, temperatura, humedad);
    archivo.close();
  } else {
    Serial.println("Error abriendo CSV");
  }

  delay(60000); // lectura cada 1 minuto
}

Lógica de Funcionamiento

El sistema opera de forma autónoma una vez energizado. A continuación se describe primero cómo interactúan físicamente los componentes del circuito y luego cómo el código gestiona ese proceso.

⚡ Funcionamiento del Circuito

Energizado Al conectar la alimentación USB, el ESP32 distribuye 3.3V al sensor DHT22 y al módulo MicroSD a través de sus pines de potencia.
Sensor El DHT22 comienza a medir temperatura y humedad del ambiente de forma continua. Cuando el ESP32 lo solicita, envía los datos digitalmente por el GPIO 4.
Módulo SD El módulo MicroSD permanece en espera. Cuando el ESP32 necesita guardar un dato, activa el pin CS en GPIO 5 y transmite la información por el bus SPI usando los pines MOSI, MISO y SCK.
WiFi La antena integrada del ESP32 se conecta a la red local para sincronizar el reloj interno con el servidor NTP, garantizando que cada registro tenga la fecha y hora real de Colombia.
Sin WiFi Si no hay red disponible, el ESP32 utiliza su reloj interno con la última hora sincronizada. El sistema no se detiene y continúa registrando datos sin interrupciones.

💻 Funcionamiento del Código

Setup Al encender el sistema se inicializa el sensor DHT22, se intenta la conexión WiFi con máximo 20 intentos, se sincroniza el reloj NTP y se prepara la tarjeta SD creando el archivo datos.csv con su encabezado si aún no existe.
Lectura Cada 60 segundos el ESP32 solicita al DHT22 los valores de temperatura en °C y humedad relativa en %. Antes de continuar valida con isnan() que los datos sean correctos y no estén corruptos.
Hora Se consulta el reloj interno del ESP32, sincronizado previamente con el servidor NTP pool.ntp.org. La fecha se formatea como YYYY-MM-DD y la hora como HH:MM:SS para compatibilidad con Excel y herramientas de análisis.
Guardado Se abre el archivo datos.csv en modo FILE_APPEND, se escribe una línea con el formato Fecha, Hora, Temperatura, Humedad y se cierra inmediatamente para proteger los datos ante cualquier corte de energía.
Espera El sistema espera 60 segundos antes de iniciar el siguiente ciclo de medición, generando exactamente un registro por minuto de forma indefinida mientras esté energizado.
